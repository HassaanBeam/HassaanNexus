# Philosophy Document Quality Checklist

Use this checklist to validate the quality and completeness of generated philosophy documents.

---

## Pre-Generation Validation

**Domain Understanding**:
- [ ] Domain is clearly defined and scoped
- [ ] Target audience identified
- [ ] Key challenges in domain understood
- [ ] Research sources identified

**Research Depth**:
- [ ] Multiple authoritative sources consulted
- [ ] Academic research included
- [ ] Industry best practices gathered
- [ ] Case studies identified

**Structure Planning**:
- [ ] All 8 core sections planned
- [ ] Domain-specific adaptations identified
- [ ] Visual elements planned (diagrams, tables)
- [ ] Length target established (1,500-3,000 words)

---

## Content Quality Standards

### Executive Summary
- [ ] Core philosophy stated in 1 clear sentence
- [ ] Problem clearly articulated
- [ ] Solution/approach explained
- [ ] Proof/evidence provided
- [ ] Readable in <2 minutes
- [ ] 100-200 words

### Foundational Principles
- [ ] 5-8 principles (not more, not less)
- [ ] Each principle has clear name
- [ ] Each principle has definition
- [ ] Each principle has research-backed rationale
- [ ] Each principle has wrong vs. right examples
- [ ] Each principle has verification test
- [ ] Examples are concrete (not abstract)
- [ ] Total 400-800 words

### Framework & Methodology
- [ ] Framework has memorable name
- [ ] Visual diagram included
- [ ] 3-5 phases defined
- [ ] Each phase has purpose
- [ ] Each phase has key activities
- [ ] Each phase has quality checklist
- [ ] Each phase has time budget
- [ ] Sequential flow is logical
- [ ] Total 400-600 words

### Anti-Patterns Dictionary
- [ ] 5-10 anti-patterns documented
- [ ] Each has memorable name
- [ ] Each has recognizable symptoms
- [ ] Each has concrete example
- [ ] Each has explanation of why it fails
- [ ] Each has actionable fix
- [ ] Each has impact metrics (if available)
- [ ] Examples are from real scenarios
- [ ] Total 400-600 words

### Design Checklists
- [ ] Pre-design checklist included
- [ ] Mid-design checklist included
- [ ] Post-design checklist included
- [ ] Items are grouped by category (3-5 groups)
- [ ] 3-5 items per group
- [ ] All items are binary (yes/no)
- [ ] All items are specific and measurable
- [ ] No ambiguous criteria
- [ ] Can be completed in <10 minutes
- [ ] Total 200-400 words

### Case Studies
- [ ] 2-3 case studies included
- [ ] Each has context and background
- [ ] Each has clear challenge/problem
- [ ] Each has approach description
- [ ] Each has "before" metrics
- [ ] Each has "after" metrics
- [ ] Each has improvement percentages
- [ ] Each has key insights
- [ ] Examples are real (not hypothetical)
- [ ] Clear connection to principles
- [ ] Total 300-500 words

### Measurement Framework
- [ ] Metrics organized in tiers (1-4)
- [ ] Each metric has target value
- [ ] Each metric has baseline value
- [ ] Red flags defined (critical issues)
- [ ] Warning signs defined
- [ ] Measurement methods explained
- [ ] Metrics are measurable
- [ ] Targets are realistic
- [ ] Total 300-400 words

### Research References
- [ ] Academic sources included
- [ ] Industry research included
- [ ] Best practice documentation included
- [ ] Organized by category
- [ ] Proper citation format
- [ ] URLs included where applicable
- [ ] Sources are authoritative
- [ ] Mix of academic and practical
- [ ] Recent sources (within 5-10 years preferred)
- [ ] Total 100-200 words

---

## Formatting Quality Standards

### Structure
- [ ] Document title follows format
- [ ] Metadata header complete
- [ ] Table of contents present
- [ ] All TOC links work correctly
- [ ] Section headers are consistent
- [ ] H1 used only for title
- [ ] H2 used for main sections
- [ ] H3 used for subsections
- [ ] H4 used sparingly (if at all)

### Visual Elements
- [ ] Code blocks use proper language tags
- [ ] Blockquotes used for purpose statements
- [ ] Tables formatted correctly
- [ ] ASCII diagrams readable
- [ ] Lists properly formatted
- [ ] Checkboxes rendered correctly
- [ ] Separators used appropriately
- [ ] Visual hierarchy is clear

### Typography
- [ ] Bold used for emphasis and labels
- [ ] Italic used sparingly
- [ ] Code formatting for technical terms
- [ ] Consistent punctuation
- [ ] No typos or spelling errors
- [ ] Consistent capitalization
- [ ] Professional tone maintained

### Navigation
- [ ] TOC matches actual sections
- [ ] Anchor links work
- [ ] Document is scannable
- [ ] Headers provide clear navigation
- [ ] Sections are logically ordered
- [ ] Cross-references work

## Completeness Standards

### Content Completeness
- [ ] No [TODO] placeholders remain
- [ ] No [PLACEHOLDER] text remains
- [ ] All sections have substantive content
- [ ] All examples are filled in
- [ ] All checklists complete
- [ ] All metrics defined
- [ ] All references cited

### Quality Completeness
- [ ] All principles are actionable
- [ ] All examples are concrete
- [ ] All frameworks are usable
- [ ] All checklists are practical
- [ ] All case studies are real
- [ ] All metrics are measurable
- [ ] All references are credible

### Professional Completeness
- [ ] Document is production-ready
- [ ] No embarrassing errors
- [ ] Professionally formatted
- [ ] Appropriate for external sharing
- [ ] Represents quality standards

---

## Readability Standards

### Clarity
- [ ] Written for target audience
- [ ] Technical terms defined
- [ ] No jargon without explanation
- [ ] Sentences are clear and concise
- [ ] Paragraphs focused on single idea
- [ ] Logical flow between sections

### Actionability
- [ ] Principles are applicable
- [ ] Frameworks are followable
- [ ] Checklists are usable
- [ ] Examples are replicable
- [ ] Tests are practical
- [ ] Guidance is specific

### Engagement
- [ ] Opening hooks reader
- [ ] Examples are relevant
- [ ] Visual breaks maintain interest
- [ ] Tone is authoritative but accessible
- [ ] Closing is memorable
- [ ] Document is not monotonous

## Evidence-Based Standards

### Research Quality
- [ ] Claims are supported by evidence
- [ ] Research is cited properly
- [ ] Sources are authoritative
- [ ] Data is credible
- [ ] Statistics are accurate
- [ ] No unsupported assertions

### Case Study Quality
- [ ] Case studies are real
- [ ] Metrics are specific
- [ ] Improvements are quantified
- [ ] Context is provided
- [ ] Results are believable
- [ ] Insights are valuable

### Framework Validation
- [ ] Framework is tested/proven
- [ ] Not purely theoretical
- [ ] Based on real practice
- [ ] Success examples exist
- [ ] Limitations acknowledged

---

## Length & Scope Standards

### Overall Length
- [ ] Total: 1,500-3,000 words (typical)
- [ ] Not too brief (<1,000 words)
- [ ] Not too verbose (>4,000 words)
- [ ] Length appropriate for domain complexity

### Section Balance
- [ ] No section dominates (>40% of total)
- [ ] No section too brief (<5% of total)
- [ ] Balance reflects importance
- [ ] Consistent depth across sections

### Scope Management
- [ ] Focused on stated domain
- [ ] Not too broad (trying to cover everything)
- [ ] Not too narrow (missing key aspects)
- [ ] Boundaries clearly defined
- [ ] Related topics acknowledged but not expanded

---

## Final Review Checklist

**Before Publishing**:
- [ ] Full document read-through completed
- [ ] All quality standards met
- [ ] Peer review conducted (if applicable)
- [ ] Test with target audience (if possible)
- [ ] Final proofread completed
- [ ] Version number assigned
- [ ] Last updated date set
- [ ] Ready for distribution

**Red Flags to Check**:
- [ ] No placeholder text remains
- [ ] No broken links or references
- [ ] No formatting errors
- [ ] No factual errors
- [ ] No contradictions between sections
- [ ] No incomplete thoughts

**Professional Standards**:
- [ ] Represents organizational quality
- [ ] Appropriate for external sharing
- [ ] Would pass expert review
- [ ] Achieves stated purpose
- [ ] Delivers value to reader

---

## Scoring Guide

### How to Use This Checklist

**Grade Categories**:
- **A (90-100%)**: Publication-ready, exceeds standards
- **B (80-89%)**: Good quality, minor improvements needed
- **C (70-79%)**: Acceptable, several improvements needed
- **D (60-69%)**: Below standard, major revisions required
- **F (<60%)**: Not ready, needs significant work

**Improvement Priority**:
1. **Critical** (Content Completeness): Must be 100%
2. **High** (Content Quality, Evidence-Based): Should be 90%+
3. **Medium** (Formatting, Readability): Should be 85%+
4. **Low** (Length, Polish): Should be 80%+

**When to Publish**:
- Minimum: B grade overall (80%+)
- Recommended: A grade (90%+)
- All Critical items: 100%

This checklist ensures philosophy documents meet high standards for quality, completeness, and professionalism.
